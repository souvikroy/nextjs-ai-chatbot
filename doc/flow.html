AI Chat Flow Architecture

// End-to-End Request Journey Through the Stack

User (U) Frontend Interface
API (C) Next.js Chat Endpoint
SDK (S) Vercel AI SDK
LLM (L) OpenAI/Anthropic
DB (D) Postgres/Drizzle
1

User Initiates Request

User (U) API (C)

The user sends a message through the UI interface, which triggers a POST request to the Next.js API endpoint with the message content and model information.

POST /api/chat
{
  message: "User query here",
  model: "gpt-4" | "claude-3",
  conversationId: "abc123"
}
2

Entitlement Check & History Fetch

API (C) Database (D)

The API handler verifies user entitlements (subscription tier, usage limits) and retrieves conversation history from the database to provide context for the LLM.

// Verify access permissions
await verifyEntitlements(userId)

// Fetch conversation context
const history = await db.messages.findMany({
  where: { conversationId }
})
3

Initialize AI Stream

API (C) Vercel SDK (S)

The API calls Vercel AI SDK's streamText function, configuring the selected model, system prompt, conversation history, and available tools (like document creation).

const stream = streamText({
  model: openai('gpt-4-turbo'),
  system: systemPrompt,
  messages: history,
  tools: { createDocument, searchWeb }
})
4

LLM Processing Begins

Vercel SDK (S) LLM (L)

The SDK forwards the request to the LLM (OpenAI/Anthropic), which begins generating the response. The LLM may produce partial text or decide to call tools based on the user's query.

LLM Response Types:
→ Partial Text (streaming tokens)
→ Tool Call (createDocument, searchWeb, etc.)
→ Function Arguments
5

Multi-Step Reasoning Loop

Vercel SDK (S) LLM (L)

The LLM can perform multi-step reasoning, iterating up to 5 times to refine its response, execute tools, and incorporate tool results back into the generation process.

Note: This iterative process allows the AI to reason through complex queries, validate information, and create structured outputs like documents or artifacts.
Step 1: Generate initial response
Step 2: Call createDocument tool
Step 3: Process tool result
Step 4: Refine response with tool output
Step 5: Final validation
6

Progressive UI Updates (SSE)

Vercel SDK (S) API (C) User (U)

The SDK streams Server-Sent Events (SSE) back through the API to the user interface. The UI progressively updates with text deltas and data chunks, creating a real-time conversation experience.

SSE Event Types:
• text-delta: Incremental text tokens
• data-kind: Metadata about response type
• data-id: Reference IDs for artifacts
• tool-call: Tool execution notifications
Real-time rendering: The artifact sidebar updates in real-time as documents are created or modified during the streaming process.
7

Tool Execution & Finalization

Vercel SDK (S) LLM (L)

If the LLM called any tools, their execution logic runs and results are fed back into the response stream. The LLM generates the final text response incorporating all tool outputs.

// Tool execution example
if (toolCall === 'createDocument') {
  const doc = await createDocument({
    title: args.title,
    content: args.content
  })
  return { documentId: doc.id }
}
8

Persistence & Cleanup

API (C) Database (D) User (U)

After the stream completes, the API persists the conversation messages and any created documents to the database. The UI clears transient loading states and finalizes the display.

// Save to database
await saveMessages({
  conversationId,
  messages: [userMessage, assistantMessage]
})

await saveDocument({
  documentId,
  userId,
  content
})
Complete: The conversation state is now fully persisted and the user can continue the dialogue or start a new thread.
